Overview

The Problem

Standard chatbot support agents handle order queries with a single prompt that embeds the entire database into the system prompt. This works at small scale — but burns 400–600 tokens on every request, including those that don't need order data at all (greetings, general questions, policy queries).

Agent 07 explores a leaner alternative: a two-stage pipeline where the first call is cheap and structural (intent extraction), JavaScript handles the zero-cost data lookup, and only the second call gets contextual data injected — keeping total tokens per turn well under 500 even with a full order record in scope.

DESIGN DECISION

Agent 07 is granted a deliberate exception to the portfolio's 1-call-per-interaction rule. The rationale: genuine tool use requires at minimum a classification pass before the lookup. The hybrid pattern is cheaper than simulated tool use (which embeds the full DB in every call) while being architecturally more honest — the reasoning trace shows real intermediate output, not staged theater.

COMPARISON

Simulated tool use (full DB in system prompt) would cost ~650 input tokens per turn regardless of query type. The hybrid pattern saves ~220 input tokens per interaction — and keeps output quality higher because Call 2 receives precise, scoped context rather than a noisy full dataset.

UI Design

Visible Reasoning Trace

The thought trace panel at the top of the chat interface shows real intermediate outputs — not staged animations. Each step corresponds to an actual pipeline event:

Step 1

Call 1 result: extracted intent type and order ID from user message, with confidence level

Step 2

JS DB lookup result: order found or not found, status surfaced immediately

Step 3

Call 2 completion: support response generated with scoped order context injected

The trace is collapsible and defaults to expanded during active processing. This makes the agentic pattern legible to technical evaluators without cluttering the experience for casual users.
