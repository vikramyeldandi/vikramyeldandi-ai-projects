01 — Problem & Objective

Why RAG over a simple FAQ bot

Agent 01 demonstrated grounding from a static policy document embedded in a system prompt. That works for short, fixed content. But a product catalogue with hundreds of SKUs, descriptions, materials, dimensions, and care instructions cannot fit in a context window — and it changes frequently.

The objective was to build a retrieval-augmented system that dynamically fetches only the relevant products at query time — making the agent scalable, accurate, and updatable without rebuilding the prompt.

The Problem

A static system prompt cannot hold a full product catalogue. It becomes outdated as inventory changes and returns irrelevant results as the catalogue grows.

The Objective

A dynamic RAG pipeline that retrieves the most relevant products at query time from a vector database, cites its sources, and answers only from what it finds.

Pipeline Flow 
// 1. User query arrives
query = "Do you have any lavender candles?"

// 2. Embed via OpenAI text-embedding-3-small
vector = openai.embed(query)  // → [0.023, -0.412, ...] 1536 dims

// 3. Semantic search in Pinecone
matches = pinecone.query(vector, topK=3)
// → CND-004 Lavender & Chamomile (score: 0.91)
// → CND-009 Eucalyptus & Mint    (score: 0.74)
// → CND-007 Oud & Rose           (score: 0.68)

// 4. Claude generates answer from retrieved context only
answer = claude.chat(system_prompt, retrieved_products, query)
// Strict rule: only answer from what was retrieved
04 — Data Preparation & Chunking

Building the vector store

The catalogue contains 100 products across 10 categories. Each product is converted into a single richly structured text chunk — combining all fields into a coherent, searchable passage. This one-product-per-chunk strategy ensures that retrieval always returns complete, coherent records rather than fragmented snippets.

Chunk Format (per product) 
Product: Lavender & Chamomile Sleep Candle (SKU: CND-004)
Category: Candles | Price: £26.00 | Stock: In Stock
Description: Calming lavender and chamomile blend formulated
to promote relaxation. Ideal for bedrooms. 45-hour burn.
Materials: 100% soy wax, cotton wick
Dimensions: H 9cm × Ø 9cm
Colours: Pale Lavender
Care: Extinguish 30 minutes before sleep.
Tags: sleep, lavender, relaxing, bedroom, soy

Each chunk is embedded using OpenAI text-embedding-3-small (1536 dimensions) and upserted into a Pinecone serverless index with full metadata stored alongside the vector — so retrieval returns the complete product record, not just an ID.

06 — Lessons Learned

What I'd do differently

1

One chunk per product is the right unit for a catalogue. Splitting product descriptions into smaller chunks (sentences or paragraphs) leads to fragmented retrievals — you get a care instruction without the price, or a colour option without the description. Full-product chunks return complete, useful records.

2

Tags and structured fields dramatically improve retrieval quality. Including tags like "sleep, lavender, bedroom, soy" means a query like "help me relax at bedtime" retrieves the right candle even without the word "candle" in the query. Semantic overlap matters.

3

Top-3 retrieval is a deliberate product decision. Returning more products increases recall but reduces answer precision — Claude starts hedging between too many options. Three is the sweet spot for a helpful, specific answer without overwhelming the user.

4

Showing retrieved sources is a trust mechanism, not just a demo feature. When users can see which products were retrieved, they understand why the answer is what it is — and they trust it more. In production, this transparency reduces support escalations.

5

The Cloudflare Worker as a multi-action proxy is a reusable pattern. Routing embed, retrieve, and generate through a single Worker endpoint — differentiated by an action field — means the browser never holds secrets, and the same infrastructure scales to Agent 03 and beyond.
